# Project File Manifest — War Galley v1.0

Last updated: 2026-03-15

## Python Engine (`Engine/`)

| File | Purpose |
|---|---|
| `__init__.py` | Package marker |
| `main.py` | TCP server; dispatches INIT_SCENARIO and PLAYER_ACTION |
| `war_galley.py` | WarGalleyEngine: turn resolution, movement, AI integration |
| `packet_parser.py` | JSON schema validation + HMAC verification |
| `security_utils.py` | SecurityManager, `--sign-all` CLI, `rotate_keys()` |
| `logger_config.py` | Centralised rotating file logger (DISA STIG AU-9) |
| `combat_resolver.py` | RAM, OAR_RAKE, national doctrines (Corvus/Ballista/Carthage) |
| `vessel_base.py` | Vessel base class: attributes, damage, stamina |
| `vessel_crew.py` | Crew: quality modifiers, fatigue, specialists |
| `vessel_factory.py` | VesselFactory: creates typed Vessel objects from JSON |
| `merchant_vessel.py` | MerchantVessel subclass: sail-driven, cargo |
| `seasonal_engine.py` | SeasonalManager: MP modifiers, storm attrition |
| `ai_logic.py` | NavalAI: autonomous vessel actions |
| `api_service.py` | Flask admin API: authenticated telemetry endpoint |
| `material_manifest_generator.py` | Generates Unity `.meta` JSON for FBX assets |
| `requirements.txt` | Pinned Python dependencies |
| `schemas/scenario_schema.json` | JSON Schema for INIT_SCENARIO packets |
| `schemas/action_schema.json` | JSON Schema for PLAYER_ACTION packets |
